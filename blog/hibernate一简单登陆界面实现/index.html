在实训期间无聊，尝试学习一些框架的东西，正好在进行数据处理，我选择了hibernate5作为我操作数据库的框架。（以前做过ssh架构下的网站的课程设计，因为专业不对口没有深入，现在做一个简单回顾也不会深入）

hibernate5简介

  • 用我自己的话来说它有几个关键词：框架，映射，对象关系，java。那它实际上是一个开放源代码的对象关系映射框架，可以对JDBC进行非常好的轻量级的封装，从而达到对象编程的思维来进行数据库操作。

具体的内容参考官方文档就OK。https://hibernate.org/orm/documentation/5.0/

项目设计

项目总体包结构

本次项目使用idea+hibernate5完成的简单的登陆注册界面功能的实现，使用的是mysql数据库

项目准备

  • idea ultimate 其中集成了很多很多框架，刚开始使用会有一定的不适，习惯后会发现它的强大。
  • hibernate5的依赖包，虽然说idea能自己下载，但资源都是国外的，所以没有代理只有自己网上下载
  • mysql8版本，作为小白，啥都喜欢最新的。
  • 当然还要有一颗耐心。。。。。。。

环境搭建

搭建问题不再描述，此博客是面向自己的，内容多描述自己遇到的问题。

问题列表

1数据库连接

idea的提供了数据库的连接，但在hibernate的配置文件中还是要对密码和用户进行定义，不然会报 “ ”@localhost password NO，等相关错误。

2建立持久化层

通过hibernate自动生成的hbm.xml文件，有可能根本没有绑定上自己的数据库，报错多为指向了空。

  • 解决方法：进入该文件对开头的table名使用idea的快捷键进行导入。

3数据库操作

hibernate的方法使用时，如save（user） 直接使用user且表设计是选择了多个类别，保存的user其中也必须包含所有的类别。

4 404错误综述

由java开发的web项目中含web.xml其作用，我的理解是完成一个映射，就是路径问题，在新建的servlet类中如若要在jsp文件中使用该类中的方法，需要jsp识别路径，路径则是在web中进行描述。或者采用注释的方法，使其得到识别。

    • web
    • java
    • jetbrain
    • hibernate5
    • idea
