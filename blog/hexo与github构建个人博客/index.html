hexo与GitHub构建个人博客

  • 1.1前言

首先了解了各种个人博客的搭建方式，我选择尝试用hexo与GitHub进行个人博客的搭建。
　　

优势如下：

1、全部是静态文件，博客访问速度

2、免费方便，不需要自己购置服务器和后台。

3、域名绑定方便。

4、数据绝对安全，由于GitHub版本管理，恢复也方便。

5、打包方便、可以转移到其他平台。

　　

1.2准备工作

如下：GitHub账号、node.js、git、hexo，工作环境我的是win10。搭建完成后还需要准备文本编辑工具，我选择的是markdownpad2。（若要配置自己的域名，当然需要购置一个域名，在博客未完善之前不建议申请域名）
　　

2.1创建GitHub仓库

命名规则为：你的用户名.GitHub.io（将来访问地址就是http://你的用户名.GitHub.io）

在仓库设置里面开启GitHub Page服务，开启服务时会选择一个主题，任意选择一个后再次开启服务既可以成功。此时就能从浏览器中访问此地址。
　　

2.2环境安装

首先下载最新的node.js~安装完成后通过cmd检验是否安装成功。

node -v
   
npm - v

若显示此命令不存在，检查环境变量是否配置完全。
　　
接下来安装git，安装完成后打开git bash进行ssh key生成。
　　

2.3ssh key绑定

　　进入你的git bash中，输入

git config --global user.name "yourname"
git config --global user.email "youremail"

这里的yourname输入你的GitHub用户名，youremail输入你GitHub的邮箱。这样GitHub才能知道你是不是对应它的账户。

当然可以验证是否输入正确

    git config user.name
    git config user.email

输入正确，就开始创建SSH，输入下面命令后，一路回车

ssh-keygen -t rsa -C "youremail"

生成SSH文件成功在电脑中找到.ssh的文件夹。路径一般为C:\Users\Administrator.ssh 内容如下：
ssh，简单来讲，就是一个秘钥，其中，id_rsa 是你这台电脑的私人秘钥，不能给别人看的，id_rsa.pub是公共秘钥，可以随便给别人看。把这个公钥放在GitHub上，这样当你链接GitHub自己的账户时，它就会根据公钥匹配你的私钥，当能够相互匹配时，才能够顺利的通过git上传你的文件到GitHub上。

而后在GitHub的setting中，找到SSH keys的设置选项，点击New SSH key
把你的id_rsa.pub里面的信息复制进去。

在git bash中，检验是否成功

ssh -T git@github.com

##3.1hexo安装
前面git和nodejs安装好后，就可以安装hexo了，你可以先创建一个文件夹blog，然后cd到这个文件夹下（或者在这个文件夹下直接右键git bash打开）。
输入命令

npm install -g hexo-cli

依旧用hexo -v查看一下版本

至此就全部安装完了。

接下来初始化一下hexo

hexo init myblog

这个blog可以自己取什么名字都行，然后

cd blog //进入这个blog文件夹
npm install

新建完成后，指定文件夹目录下有：

  • node_modules: 依赖包
  • public：存放生成的页面
  • scaffolds：生成文章的一些模板
  • source：用来存放你的文章
  • themes：主题
  • ** _config.yml: 博客的配置文件**

输入以下命令，就可以通过静态生成，在网站输入locahost：4000就能看到你搭建的博客了。

hexo g 
hexo sever

生成的博客大概就是这样的，里面会有hexo提供的一片hello文章。
##3.2将hexo部署到GitHub
通过上面这些步骤我们只是在本地搭建起了博客，要让别人看到还需要把它部署到GitHub上。

之前我们已经创建好了我们的GitHub仓库，用户名是：你的用户名.GitHub.io。

同时我们还生成了电脑的SSHkey，此时GitHub与自己的电脑本地建立了免密访问。
现在到你的本地安装hexo的文件中找到hexo的配置文件_config.yml，在里面找到deploy关键词进行如下配置：

deploy:
    type: git
    repo: https://github.com/hhs44/hhs44.github.io.git
    branch: master

这个时候需要先安装deploy-git ，也就是部署的命令,这样你才能用命令部署到GitHub。

npm install hexo-deployer-git --save
hexo clean
hexo generate
hexo deploy

设置完成后，部署博客内容就不需要输入密码了。
##3.3第一篇博客
到这里就可以开始写自己的第一篇博客了。当然在写之前需要准备编辑工具，我使用的是MarkdaownPad2，可以一边编辑一边查看编辑后的效果，以及浏览器预览等。

新建一篇博客需要通过cmd或者Git bash进入到你的本地博客，并输入命令

hexo new  <title> title是你的文章标题

此时在E:\blog\source_posts中就会生成对应的md文件，使用编辑器进行编辑并保存后，通过以下命令即可发布到你的个人博客上了。

hexo g
hexo d

基础篇终于告一段落，后面我会就每次优化自己的博客进行记录，若有错误的地方请大家帮忙指出。—————— 评论功能还在进行中。。。
