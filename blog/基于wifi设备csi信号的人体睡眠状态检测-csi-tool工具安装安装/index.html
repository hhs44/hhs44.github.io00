安装构建工具，Linux开发头和Git客户端

sudo apt-get install gcc make linux-headers-$(uname -r) git-core

  • 获取CSI Tool Linux源代码树，其中包括对无线驱动程序的修改，并在存储库中检查上游内核版本的相应标记：（若从git仓库下载以下命令才有意义）

    echo CSITOOL_KERNEL_TAG = csitool-$(uname -r | cut -d. -f 1-2)

    git checkout ${CSITOOL_KERNEL_TAG}

  • 为现有内核构建修改后的驱动程序：

    make -C /lib/modules/$(uname -r)/build M=$(pwd)/drivers/net/wireless/iwlwifi modules

  • 将修改后的驱动程序安装到模块更新目录中：

    sudo make -C /lib/modules/$(uname -r)/build M=$(pwd)/drivers/net/wireless/iwlwifi INSTALL_MOD_DIR=updates modules_install

安装修改后的固件

  • 获取CSI工具补充材料

git clone https://github.com/dhalperi/linux-80211n-csitool-supplementary.git

  • 重新定位英特尔Wi-Fi Link 5000系列适配器的所有现有固件：

    for file in /lib/firmware/iwlwifi-5000-*.ucode; do sudo mv $file $file.orig; done

  • 安装修改后的固件：

    sudo cp linux-80211n-csitool-supplement/firmware/iwlwifi-5000-2.ucode.sigcomm2010/lib/firmware

    sudo ln -s iwlwifi-5000-2.ucode.sigcomm2010 /lib/firmware/iwlwifi-5000-2.ucode

构建用户空间日志记录工具

  • 构建log_to_file，这是一个命令行工具，它将通过驱动程序获得的CSI写入文件：

    make -C /home/huang/桌面/课程设计/linux-80211n-csitool-supplementary/netlink

安装完成后，就可以使用CSI Tool记录CSI信号了

  • 首先，我们需要卸载原来的驱动，然后再以打开记录CSI的模式加载驱动：

    sudo modprobe -r iwlwifi mac80211

    • 卸载驱动程序后，在启用CSI日志记录的情况下重新加载它：

    ``sudo modprobe iwlwifi connector_log=0x1`

  • 首先，确保网卡处于开启状态：（wlan0为自己的网卡名，通过ifconfig查询）

    sudo ip link set wlan0 up

  • 然后，扫描搜索的WiFi：

    sudo iw dev wlan0 scan | less

  • 最后，连接所需要的WiFi：

    sudo iw dev wlan0 connect [网络SSID]

注意，这里的WiFi网络要求是没有密码的！

通过使用（例如）iw和iproute2实用程序（或者如果为无线接口启用NetworkManager，使用nm-cli或图形小程序）连接到802.11n接入点。或者，您可以通过安装和配置hostapd让您的系统作为802.11n接入点运行（请参阅配置示例的补充材料）; 在这种情况下，请在此系统上启动hostapd，然后将另一个802.11n工作站连接到该系统。

  • 连接网络后，使用如下命令将CSI数据记录到文件中：

    sudo ～/linux-80211n-csitool-supplementary/netlink/log_to_file csi**.**dat

  • 下面，在同一WiFi网络中的另一终端上ping已装好CSI Tool的主机，IP地址可以使用ifconfig查到。在多次ping之后，就可以得到CSI数据了。
